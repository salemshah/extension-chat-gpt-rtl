# Changelog

All notable changes to ChatGPT RTL Support are documented here.  
Format follows [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

---

## [1.0.0] — 2026-05-09

### Added

- Auto-detection of Persian, Arabic, Hebrew, and Urdu using Unicode character
  ratio (≥ 30 % RTL strong chars → RTL classification)
- Per-paragraph and per-list-item direction detection inside assistant messages
- Real-time input area direction switching as you type
- MutationObserver (debounced 160 ms + `requestIdleCallback`) for ChatGPT's
  streaming SPA
- Popup UI with Enable/Disable toggle, Auto-detect toggle, Force RTL / LTR
  buttons, and Reset
- Settings persisted and synced via `chrome.storage.sync`
- **Keyboard shortcuts** active on the ChatGPT tab:
  - `Alt+Shift+R` — Force RTL
  - `Alt+Shift+L` — Force LTR
  - `Alt+Shift+A` — Return to Auto mode
- Code blocks (`<pre>`, `<code>`) always pinned to `dir="ltr"` — JS + CSS
  double-protection
- Copy buttons, action menus, sidebars, modals, and dialogs are explicitly
  excluded from any direction override
- Cross-browser storage shim: works in Chrome (callback API) and Firefox
  (Promise API) from the same source
- `browser_specific_settings.gecko` in `manifest.json` for Firefox
  sideloading compatibility
- Real PNG icons (16 × 16, 48 × 48, 128 × 128) generated by `generate_icons.py`
  using only Python standard library
- `safeQSA` / `safeClosest` wrappers so unknown future selectors fail silently
  instead of crashing the extension
- Semantic-first selector strategy: `[data-message-author-role]`, `article`,
  `main article` before any class-name fallback
- Privacy: zero network requests, no analytics, only `storage` permission
